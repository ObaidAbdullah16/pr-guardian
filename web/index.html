What gets checked

Seven automated checks that catch the most common PR mistakes.

📄

README Updated

Was README.md touched in this PR? New features need documentation.

🔗

Issue Linked

Does the PR description reference an issue with Fixes #123 or Closes #?

📝

Changelog Updated

Was CHANGELOG.md updated so users know what changed in each release?

📋

Contributor Guide

Does a CONTRIBUTING.md exist so contributors know the rules of the repo?

✍️

PR Description

Is the PR description meaningful? Requires at least 30 characters of context.

🌿

Branch Name

Does the branch follow convention? (feat/, fix/, docs/, chore/)

🔍

Broken Links

Are all Markdown links in the PR description reachable? Checks with HTTP HEAD.

Install in 60 seconds

Add this file to your repo. That's it.

Step 1

Create .github/workflows/pr-guardian.yml in your repo:

.github/workflows/pr-guardian.yml 
name: PR Guardian

on:
  pull_request:
    types: [opened, synchronize, reopened]

permissions:
  pull-requests: write
  contents: read

jobs:
  pr-guardian:
    runs-on: ubuntu-latest
    steps:
      - uses: obaid/pr-guardian@v1
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          fail_on_error: "false"   # set to "true" to block merging
Step 2

Commit and push. PR Guardian will comment on every new PR automatically. ✅
